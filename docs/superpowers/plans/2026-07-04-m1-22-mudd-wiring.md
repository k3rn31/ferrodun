# M1-22 `mudd` Single-Process Wiring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Boot registered tenants end-to-end in one `mudd` process — world load, DB, scheduler driver loop, command pipeline, embedded gateway — so `cargo run -p mudd -- --tenant-dir <dir>` serves a telnet login, and a registry with two tenants serves both concurrently.

**Architecture:** `PersistentWorld` (mud-db) takes ownership of the `Scheduler` and becomes the single write path (arena-first, durable-write-second, fail-stop on `DbError`); the pipeline stops mutating the world and returns effects instead. `mudd` boots one isolated world+gateway stack per tenant (in-memory IPC pair, one World task + one gateway task each), configured by a server-wide XDG config with a `[[tenants]]` registry.

**Tech Stack:** Rust (edition 2024), tokio, clap, figment, sqlx/SQLite, existing workspace crates (`mud-core`, `mud-db`, `mud-engine`, `mud-gateway`, `mud-ipc`, `mud-world`, `mud-i18n`, `mud-schema`, `mud-account`, `mud-net`).

**Design doc:** `docs/superpowers/specs/2026-07-03-m1-22-mudd-wiring-design.md` — read it before starting.

## Global Constraints

- `unwrap()` forbidden everywhere; `expect()` only in tests, with a descriptive message. No `panic!`/`todo!`/`unreachable!` in production code.

```rust
    /// Removes and returns every queued command in arrival order, advancing
    /// the tick counter once (saturating at [`u64::MAX`]).

    /// This is the seam for a durable applier that owns the write path
```

```rust
use mud_schema::WorldId;

impl TenantDb {
    /// This tenant's stable World identity (§2.1.3.2), generated at first call
    /// and persisted; every later call — including after a restart — returns
    /// the same value.
    ///
    /// # Errors
    ///
    /// Returns [`DbError`] on a query failure or a corrupt persisted id.
    pub async fn world_id(&self) -> Result<WorldId, DbError> {
        // Random positive i64; NonZeroU64 rules out 0, the CHECK-ed single
        // row rules out a second value ever being written.
        let fresh = i64::from(rand::random::<u32>()).saturating_add(1);
        sqlx::query!(

            fresh

        .execute(&self.pool)
        .await?;

            .fetch_one(&self.pool)
            .await?;
        let raw = u64::try_from(row.world_id).map_err(|_| DbError::InvalidId(row.world_id))?;
        let value = NonZeroU64::new(raw).ok_or(DbError::InvalidId(row.world_id))?;
        Ok(WorldId::new(value))
    }
}
```

```rust
    /// Enqueues `command` for the next [`tick`](PersistentWorld::tick)
    /// (§2.5.3.5: arrival order is application order).
    pub fn submit(&mut self, command: MutationCommand) {
        self.scheduler.submit(command);
    }

    /// Drains the scheduler queue, applying each command to the arena and the
    /// database via [`apply`](PersistentWorld::apply), and returns the
    /// observable events (created handles, precondition failures, rejections).
    ///
    /// # Errors
    ///
    /// Returns the first [`DbError`] and stops: commands after the failure are
    /// neither applied nor persisted (fail-stop, the M1-22 consistency model —
    /// the caller must halt dispatch and exit).
    pub async fn tick(&mut self) -> Result<Vec<TickEvent>, DbError> {
        let mut events = Vec::new();
        for command in self.scheduler.drain() {
            if let Some(event) = self.apply(command).await? {
                events.push(event);
            }
        }
        Ok(events)
    }

    /// The current tick number (source for `mud.time.tick()`, §3.16.4).
    #[must_use]
    pub fn tick_number(&self) -> u64 {
        self.scheduler.tick_number()
    }
```

```rust
    /// Makes a persisted entity resident in the live world (M1-22 design §7):
    /// mints an arena handle for `key`, replays its persisted location, and
    /// records the key↔id mapping. Called for entities created after the boot
    /// load — e.g. a puppet minted mid-session by the login flow (§3.19).
    /// Idempotent: an already-resident key returns its existing handle.
    ///
    /// # Errors
    ///
    /// Returns [`DbError`] if `key` has no `entities` row, on a query failure,
    /// on a persisted slug naming no loaded room, or on arena exhaustion.
    pub async fn hydrate(&mut self, key: EntityKey) -> Result<EntityId, DbError> {
        if let Some(id) = self.by_key.get(&key) {
            return Ok(*id);
        }
        let key_db = entity_key_to_db(key)?;
        let exists = sqlx::query!(
            r#"SELECT entity_key AS "entity_key!" FROM entities WHERE entity_key = ?"#,
            key_db
        )
        .fetch_optional(self.db.pool())
        .await?;
        if exists.is_none() {
            return Err(DbError::DanglingReference(key_db));
        }

        let id = self
            .world
            .create()
            .map_err(|source| DbError::LoadArenaExhausted { entity_key: key, source })?;
        self.by_key.insert(key, id);
        self.by_id.insert(id, key);

        let location = sqlx::query!(
            r#"SELECT place_key AS "place_key!" FROM location WHERE entity_key = ?"#,
            key_db
        )
        .fetch_optional(self.db.pool())
        .await?;
        if let Some(row) = location {
            let place = place_id_for_slug(&self.places, &row.place_key)?;
            self.world
                .move_to(id, place)
                .map_err(|_| DbError::DanglingReference(key_db))?;
        }
        Ok(id)
    }
```

```rust
pub async fn run(
    mut endpoint: InMemoryEndpoint<WorldFrame, GatewayFrame>,
    world_id: WorldId,
    world: Arc<Mutex<PersistentWorld>>,
    backend: DbBackend,
    mut sessions: SessionService,
    mut pipeline: Pipeline,
    builtins: Vec<Command>,
    places: WorldPlaces,
) -> anyhow::Result<()> {
    let live = accept_resume(&mut endpoint, world_id).await.context("ipc resume handshake")?;
    anyhow::ensure!(live.is_empty(), "a fresh gateway must announce no live sessions");

    let mut ticker = tokio::time::interval(TICK_PERIOD);
    ticker.set_missed_tick_behavior(MissedTickBehavior::Delay);
    loop {
        tokio::select! {
            _ = ticker.tick() => {
                // Fail-stop on DbError: ? ends the loop, run() returns Err,
                // main exits non-zero, the supervisor restarts (design §8).
                let events = world.lock().await.tick().await.context("durable tick")?;
                for event in events {
                    tracing::warn!(?event, "tick event"); // Created is info-worthy; match and split levels
                }
            }
            frame = endpoint.recv() => match frame.context("ipc recv")? {
                None => return Ok(()), // gateway closed cleanly
                Some(GatewayFrame::Connect(connect)) => {
                    for output in sessions.connect(connect.session_id) {
                        endpoint.send(WorldFrame::Output(output)).await.context("send output")?;
                    }
                }
                Some(GatewayFrame::Disconnect(disconnect)) => sessions.disconnect(disconnect.session_id),
                Some(GatewayFrame::Input(input)) => {
                    handle_input(&mut endpoint, &world, &backend, &mut sessions, &mut pipeline, &builtins, &places, input).await?;
                }
                Some(GatewayFrame::Resume(_)) => tracing::warn!("unexpected mid-stream resume frame dropped"),
            }
        }
    }
}
```

`handle_input` (private): route via `sessions.on_input(input.session_id, input.line.as_str(), &backend).await`:
- `Routing::Login { outputs, close }` → send each output; if `close`, send `WorldFrame::Close(SessionClose { session_id })` and `sessions.disconnect(session_id)`.
- `Routing::InWorld` → lock the world **after** `on_input` returned (never across it): `let mut guard = world.lock().await;` then `pipeline.dispatch(guard.world(), &places, &sessions.resolver(&builtins), &input)`; on `Ok(outcome)`: submit each effect (`guard.submit(MutationCommand::new(effect))`), drop the guard, send outputs; on `SessionDisposition::Close` send the `Close` frame + `sessions.disconnect`. On `Err(PipelineError::UnknownSession(_))` log warn and drop; `Err(CommandIdExhausted)` → propagate as fatal (`anyhow`).
- `Routing::Unknown` → `tracing::warn!` and drop.

Match `SessionDisposition`'s exact variants (read `crates/mud-engine/src/dispatch.rs`); split tick-event log levels: `TickEvent::Created` at `info`, `PreconditionFailed`/`Rejected` at `warn`.

```markdown
# Running a server

## Quick start
    mudd --tenant-dir /path/to/tenant
(serves telnet on 127.0.0.1:4000)

## Server configuration
Path: $XDG_CONFIG_HOME/ferrodun/config.toml (default ~/.config/ferrodun/config.toml),
override with --config. All keys optional.

    rate = 10        # per-session sustained commands/second
    burst = 20       # per-session burst allowance

    [[tenants]]      # the tenant registry: one block per tenant
    dir = "/srv/ferrodun/tenants/midgard"
    listen = "127.0.0.1:4000"

    [[tenants]]
    dir = "/srv/ferrodun/tenants/asgard"
    listen = "127.0.0.1:4001"

Each tenant is an isolated stack (own database, own world, own listener).
Listen addresses must be distinct; each tenant's `config.toml` must carry a
`tenant_tag` unique across the registry.

Layering, weakest first: built-in defaults < config.toml < MUDD_* environment
variables (e.g. MUDD_RATE=5) < CLI flags. --tenant-dir replaces the whole
registry with a single tenant on --listen (default 127.0.0.1:4000).

## Per-tenant configuration
<tenant-dir>/config.toml is the world's file: start_room (required),
tenant_tag (default 0), locale (default "en"), banner/palette paths.

## Running under a supervisor (Linux)
mudd is fail-stop by design: on an unrecoverable error (database write
failure, IPC loss) it stops serving immediately and exits non-zero. State is
rebuilt from the database at boot, so the recovery mechanism is "restart the
process" — run it under a supervisor.

    # /etc/systemd/system/ferrodun.service
    [Unit]
    Description=Ferrodun MUD server
    After=network.target

    [Service]
    ExecStart=/usr/local/bin/mudd
    Restart=on-failure
    RestartSec=2
    # A persistent fault (e.g. full disk) must not crash-loop forever:
    StartLimitIntervalSec=60
    StartLimitBurst=5

    [Install]
    WantedBy=multi-user.target

Container runtimes: use `restart: on-failure` (compose) or an equivalent
restart policy.
```
